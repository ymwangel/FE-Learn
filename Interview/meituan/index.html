二：DOM的attribute属性和property属性区别

  • 1. DOM有其默认的基本属性，而这些属性就是所谓的property，例如：id、value等 无论如何，它们都会在初始化的时候在DOM对象上创建
  • 2. HTML标签中自定义的属性和值都会保存在该DOM对象的attributes属性里面。这些attribute属性的 javascript中的类型是attr，例如：sth属性
  • 3. property能够从attributed中得到同步
  • 4. attribute不会同步到property上的值
  • 5. attribute和property之间的数据绑定是单向的，attribute > property
  • 6. 更改property和attribute上的任意值，都会将更新反映到HTML页面中

三：浏览器内核

  • 1. 浏览器内核分为：渲染引擎（Layout engineer 或者 Rendering Engine）、 JS引擎

  • 3. JS引擎：解析javascript语言，执行javascript语言来实现网页的动态效果

四：img的alt属性和title属性区别：

  • 1. alt用于图片没有显示时在图片显示区域显示一个说明文字
  • 2. title表示鼠标在图片上停留时，显示一个悬浮框，显示其中的文字

五：viewport常用的属性设置：移动端

  • 1. width：控制viewport的大小，可以指定一个值，如果600，或者特殊的值，如device-width为设备的宽度（单位为缩放为100%时的css像素）
  • 2. height：和width相对应，指定高度
  • 3. initial-scale：初始缩放比例，也就是当页面第一次load时的缩放比例
  • 4. maximum-scale：允许用户缩放到的最大比例
  • 5. minimum-scale：允许用户缩放到的最小比例
  • 6. user-scalable：用户是否可以手动缩放

六：strong 和 em的区别： 都是表达要素

  • 1. strong 表示html页面上的强调：
    • (1). 重要性：可用于标题、说明、或者段落上、来显示文字的重要性
    • (2). 严重性：可被用来标记警告或者警示标志
    • (3). 紧急性：可以被用来表示需要被尽快看见的部分
  • 2. em：表示句子中的强调（即强调语义）
  • 3. strong标签的文字，加粗
  • 4. em标签的文字为斜体

十：px、em、rem、vh、vw、vmin、vmax

  • 1. px：相对长度单位，像素px是相对于显示器屏幕分辨率而言的
  • 2. em：相对长度单位。相对于当前对象内文本的字体尺寸。如当前对行内文本的字体尺寸未被人为设置，则相对于浏览器的默认字体尺寸
  • 3. rem：相对长度单位。r：root，相对于根元素 html 的字体的大小

  • 4. vh、vw：相对于视口的高度和宽度，而不是元素的（CSS百分比是相对于包含它的最近的父元素的高度和宽度）

    • 1vh 等于 1/100的视口高度，1vw等于1/100的视口宽度
  • 5. vmin和vmax：关于视口高度和宽度的两者的最小值和最大值

十一：display:none 与 visibility:hidden 还有 opacity:0 的区别

  • display:none: 不为被隐藏的对象保留其物理空间，即该对象在页面上彻底消失，看不见也摸不到
  • visibility:hidden 使对象在网页上不可见，但该对象在网页上所占的空间没有改变，看不见但摸得到
  • opacity:0 ： 设置为透明，

四：null、undefined的区别

  • null: 表示空值, typeof(null) 为“object”
  • undefined：未定义
  • Object.prototype.toString.call(null) // "[object Null]" Object.prototype.toString.call(undefined) // "[object Undefined]"

五：call、apply的区别

  • call: 第一个参数是作为函数上下文的对象，后面传入的是一个参数列表，而不是单个数组
  • apply：两个参数：一个是作为函数上下文的对象，另外一个是作为函数参数组成的数组

五：cookie、SessiongStorage、LocalStorage的区别

  • cookie: 是存储在客户端的小型文本文件，可以包含若干键值对，cookie会在每次发送HTTP请求时 附加到cookie 头字段，服务器以此得知 用户所处的状态
  • SessionStorage、LocalStorage：都是webStorage，都是本地存储，不会被发送到浏览器上，只是SessionStorage，在浏览器会话结束时清空，，而LocalStorage，没有过期时间，除非手动清除
